Optimizing Cloud Connectivity with Azure ExpressRoute

📅 September 2023 ⏱ 6 min read 👤 Shawqy Ahamed

When an enterprise commits to Azure as its cloud platform, the connectivity between on-premises infrastructure and Azure becomes a critical path dependency. Internet-based connections work for low-bandwidth, latency-tolerant workloads, but for SAP HANA, SQL Managed Instance, or latency-sensitive financial applications, you need dedicated connectivity — and that means ExpressRoute.

ExpressRoute delivers consistent sub-10ms latency and predictable bandwidth that no Internet-based connection can match. For workloads where performance SLAs matter, it is not optional.

ExpressRoute Fundamentals

ExpressRoute provides a private, dedicated connection between your on-premises network and Microsoft's global network, bypassing the public Internet entirely. The connection is established through an ExpressRoute partner — a telco or network service provider — at one of Microsoft's peering locations.

There are two peering types you'll typically configure: Azure Private Peering provides connectivity to Azure virtual networks. Microsoft Peering provides connectivity to Microsoft public services like Microsoft 365 and Dynamics 365.

Designing for Redundancy

A single ExpressRoute circuit is not sufficient for business-critical workloads. The correct architecture for 99.99% availability is dual circuits from different providers, connecting to different peering locations, with BGP configured for automatic failover.

BGP Configuration for Active-Active

Configure both circuits in active-active mode with equal BGP weight and local preference. Use AS-PATH prepending to prefer one circuit for outbound traffic while maintaining the other as an active hot standby.

  • Primary circuit: Default BGP weight (100), no AS-PATH prepending
  • Secondary circuit: Default BGP weight (100), AS-PATH prepend x1 for outbound preference
  • Configure BFD (Bidirectional Forwarding Detection) for sub-second failure detection
  • Set hold-time to 90 seconds and keepalive to 30 seconds on BGP sessions

Performance Optimization

Use ExpressRoute Global Reach to connect your on-premises sites to each other through Microsoft's backbone. Enable FastPath on your ExpressRoute Gateway to bypass the gateway for data-path traffic, reducing latency for high-throughput workloads.

The Results

In the Azure WAN optimization project I led, dual ExpressRoute circuits with BGP route optimization and FastPath delivered a 40% reduction in average round-trip latency. We achieved 99.99% connectivity uptime over 18 months — with two transparent failover events that users never noticed.
